# KwikSnax Redux — Core Loop

## Игровая сессия

Игровая сессия представляет собой последовательность уровней, которые игрок проходит до поражения или завершения игры.

Типичная сессия включает:

- Запуск игры
- Выбор режима (одиночная игра / кооператив)
- Начало уровня
- Прохождение уровня
- Переход к следующему уровню при успехе
- Потеря жизни при ошибке
- Завершение игры при потере всех жизней или выполнении условий победы
- Возможность немедленного перезапуска

Сессии должны быть короткими и повторяемыми.
Игрок должен иметь возможность быстро начать новую попытку без длительных загрузок или пауз.

## Основной цикл

Основной игровой цикл строится вокруг повторяющейся последовательности действий игрока:

- Анализ текущей ситуации на уровне
- Перемещение персонажа по игровому полю
- Взаимодействие с объектами уровня
- Избегание врагов и опасностей
- Уничтожение врагов или выполнение условий уровня
- Очистка уровня
- Переход к следующему уровню

Во время игры игрок постоянно принимает быстрые тактические решения:

- куда двигаться
- когда атаковать
- когда избегать опасности
- как эффективнее очистить уровень
- как минимизировать риск

Основной цикл является:

- быстрым
- читаемым
- предсказуемым
- пригодным для повторения много раз подряд

## Прогресс

Прогресс в игре происходит в рамках одной игровой сессии.

Основные элементы прогресса:

- последовательное прохождение уровней
- постепенное увеличение сложности
- увеличение количества врагов
- изменение конфигурации уровней
- ускорение игрового темпа
- рост требований к точности игрока

Дополнительные элементы прогресса:

- Подсчёт очков включён по умолчанию для всех игровых режимов.
- Бонус за скорость применяется, если время прохождения уровня ниже контрольного времени уровня. Бонус рассчитывается как X очков за каждую сэкономленную секунду.
- Бонус за эффективную игру применяется, если игрок использовал ≤ Y действий или ≤ Z ресурсов. Бонус даёт фиксированный множитель или фиксированное количество очков.
- Таблица рекордов активна, когда включён онлайн-режим. Таблица заполняется только данными аутентифицированных игроков. При равных результатах порядок определяется временем достижения результата.

Прогресс не усложняет базовые правила игры.
Усложнение происходит за счёт скорости, количества объектов и взаимодействия механик, а не за счёт введения новых сложных систем.
