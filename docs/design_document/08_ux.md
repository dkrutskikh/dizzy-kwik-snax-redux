# KwikSnax Redux — UX

## Screens

The interface consists of a minimal number of screens and provides direct access to gameplay.

Main screens:

- main screen
- mode selection screen
- gameplay screen
- pause screen
- level completion screen
- defeat screen
- results screen

Requirements:

- transitions between screens must be fast
- long loading animations are not allowed
- the player must be able to start the game with minimal actions
- restarting a level must be instant

On dual-screen systems:

- the main screen is used for the playfield
- the secondary screen is used for information, menus, or statistics

Menus must have consistent controls across all modes.


## HUD

The HUD must be minimal and not interfere with gameplay.

Rules:

- the playfield has priority over the interface
- the HUD must be readable at a glance
- important information must always be visible
- HUD elements must not overlap gameplay objects

The HUD displays:

- score
- lives
- level number
- bonuses
- player status
- second player status

On dual-screen systems, the HUD is placed on the secondary screen.

State changes are accompanied by:

- short animations
- sound cues
- color changes


## Readability

Gameplay must remain readable at high speed.

Main requirements:

- objects must be distinguishable by shape
- objects must be distinguishable by color
- dangerous objects must be immediately noticeable
- the player can locate their character by silhouette and color
- the background must not interfere with gameplay objects

Readability rules:

- avoid small details
- avoid colors that are too similar in brightness
- do not overload the screen with objects
- avoid blinking elements except for defined feedback cases

All important events must be visible without stopping player movement.


## Learning

The game must be understandable without long instructions.

Main principles:

- early levels teach through gameplay
- new mechanics are introduced gradually
- the player can infer the rules without text
- mistakes are indicated by clear feedback

Allowed methods:

- short hints
- simple icons
- visual signals
- demonstration of mechanics within the level

Not allowed:

- long blocks of text
- stopping the game for extended periods
- requiring reading before starting gameplay

The player must be able to start playing immediately and understand the rules through experience.
