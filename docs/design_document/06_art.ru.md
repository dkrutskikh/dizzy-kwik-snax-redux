# KwikSnax Redux — Art

## Общий стиль

Визуальный стиль игры должен соответствовать классическим портативным и аркадным играм конца 90-х годов.

Основные принципы стиля:

- простая и читаемая графика
- чёткие силуэты персонажей и объектов
- минимальное количество мелких деталей
- высокая контрастность между игровыми элементами
- отсутствие визуального шума

Графика должна хорошо читаться на небольшом экране и при быстром игровом темпе.

Стиль сочетает:

- ретро-эстетику
- мультяшную выразительность
- яркие цвета
- понятные формы

Анимации должны быть узнаваемы по силуэту или движению. Максимальная длительность зацикленных и игровых анимаций: 200 ms. Максимальное число ключевых кадров на анимацию задаётся в пайплайне (например, 8 для читаемости силуэта). Короткие одноразовые эффекты могут быть короче (например, 67–100 ms).

## Палитры

Цветовые палитры должны учитывать ограничения портативных консолей.

Основные правила:

- не более 4 цветов на спрайт; не более 16 цветов на аппаратную палитру
- использование заранее определённых палитр
- минимальный контраст 4.5:1 между игровым передним планом и фоном
- одинаковые цвета сопоставляются именованной семантической таблице для однозначной проверки

Разделение палитр:

- отдельные палитры для персонажей
- отдельные палитры для врагов
- отдельные палитры для фона
- отдельные палитры для бонусов и опасностей

Изменение палитры используется для:

- обозначения сложности уровня
- отображения состояния объекта
- визуальной индикации бонусов
- различия между игроками в кооперативе

## Персонажи и объекты

Все игровые объекты различимы по форме и цвету.

Требования к персонажам:

- чёткий силуэт
- хорошо читаемая анимация
- различимые направления движения
- различимые состояния (движение, ожидание, поражение)

Требования к врагам:

- визуально отличаться от игрока
- иметь понятное направление движения
- иметь заметные анимации атаки или опасности

Требования к объектам:

- предметы для сбора должны выглядеть безопасными и полезными
- опасные объекты должны выглядеть опасными
- бонусы должны отличаться цветом или формой
- интерактивные элементы должны быть заметны

Размеры спрайтов должны соответствовать возможностям целевой платформы и не создавать перегрузку экрана.

## UI / HUD

Интерфейс должен быть минималистичным и не мешать игровому процессу.

Основные правила:

- игровое поле занимает большую часть экрана
- UI читаем с первого взгляда
- важная информация должна быть всегда видна
- не допускается перекрытие игровых объектов интерфейсом

HUD отображает:

- очки
- количество жизней
- номер уровня
- бонусы
- состояние игроков

На системах с двумя экранами:

- основной экран используется для игрового поля
- дополнительный экран используется для HUD, статистики или служебной информации

Анимации интерфейса — не более 200 ms и не отвлекают от игры.
