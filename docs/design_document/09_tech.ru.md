# KwikSnax Redux — Tech

## Движок

Игра разрабатывается на движке ToyGine.

Требования к движку:

- работа на реальном железе без эмуляции
- одинаковая логика на всех платформах
- фиксированный шаг симуляции
- минимальный input lag
- детерминированное поведение
- поддержка 2D-спрайтовой графики
- поддержка тайловых фонов
- поддержка ограниченного количества каналов аудио
- возможность сборки под разные платформенные профили

Движок должен позволять:

- быстро запускать игру на целевой платформе
- тестировать игру в эмуляторе и на железе
- включать / отключать функции через профили
- измерять производительность

Игра должна использовать только те возможности движка,
которые доступны на всех целевых платформах.

## Платформенные профили

Сборка должна выполняться с использованием платформенных профилей.

Профиль определяет:

- разрешение экрана
- количество цветов
- размер спрайтов
- количество объектов на экране
- частоту обновления
- количество каналов аудио
- доступную память
- доступные устройства ввода

Каждая платформа должна иметь отдельный профиль.

Профили должны позволять:

- собирать игру без изменения кода
- ограничивать использование ресурсов
- проверять совместимость со слабой платформой

Основной профиль должен соответствовать самой слабой поддерживаемой системе.

## Ограничения (самой слабой платформы)

Дизайн игры должен учитывать ограничения самой слабой платформы.

Ограничения могут включать:

- маленькое разрешение экрана
- ограниченное количество цветов
- ограниченное количество спрайтов
- небольшое количество памяти
- ограниченная скорость CPU
- ограниченное количество каналов аудио
- медленная загрузка данных

Правила:

- игра должна работать стабильно на слабой платформе
- все уровни должны быть проходимы на слабой платформе
- интерфейс должен помещаться на маленьком экране
- количество объектов должно быть ограничено
- анимации должны быть короткими

Если функция не работает на слабой платформе,
она не должна быть обязательной для игры.

## Поддерживаемые платформы (план)

Планируемые платформы:

- Nintendo GBA
- Nintendo DS
- PC (Windows, Linux, macOS)

Дополнительно возможно:

- Nintendo 3DS
- Nintendo Switch
- Nintendo GameCube
- Nintendo Wii / Wii U
- будущие портативные системы

Требования:

- игра должна работать на реальном железе
- управление должно соответствовать консоли
- частота обновления должна быть стабильной
- не допускается зависимость от PC-специфичных функций

Поддержка новых платформ не должна ломать существующие.

## Технические цели

Основные технические цели проекта:

- стабильная частота кадров
- минимальный input lag
- одинаковый игровой тайминг на всех платформах
- быстрая загрузка уровней
- малый размер сборки
- малое потребление памяти
- стабильная работа на реальном железе
- отсутствие случайных зависаний

Дополнительные цели:

- возможность записи повторов
- возможность детерминированного воспроизведения
- возможность автоматических тестов
- возможность измерения производительности
- возможность CI-сборок под все платформы

Техническое качество считается важнее визуальных эффектов.
