# KwikSnax Redux — Audio

## Музыкальный стиль

Музыкальное сопровождение должно соответствовать стилю классических аркадных и портативных игр конца 90-х годов.

Основные требования:

- короткие зацикленные композиции
- чёткий ритм, поддерживающий быстрый игровой темп
- простая, запоминающаяся мелодия
- ограниченное количество инструментов
- хорошая читаемость на слабых динамиках портативных устройств

Музыка должна:

- поддерживать динамичный игровой процесс
- не отвлекать от игрового процесса
- оставаться чёткой при зацикливании
- соответствовать визуальному стилю игры

Разные темы используются для:

- обычных уровней
- сложных уровней
- бонусных ситуаций
- завершения уровня
- поражения

## Звуковые эффекты

Звуковые эффекты должны быть короткими, чёткими и различимыми по высоте или паттерну.

Основные правила:

- каждый тип действия имеет свой звук
- звук распознаётся игроком без путаницы
- звук не перекрывает другие важные звуки
- длительность звуков минимальна

Обязательные категории звуков:

- перемещение
- сбор предмета
- уничтожение врага
- получение бонуса
- получение урона
- потеря жизни
- завершение уровня
- появление опасности
- события интерфейса

Звуки должны учитывать ограничения целевой платформы. Используются следующие числовые бюджеты, если не задано платформенное переопределение:

- Максимум одновременных каналов: 8 (настраивается по платформе; для mobile/tablet/console могут быть меньшие значения, например 4–6).
- Частота дискретизации: 22050 Hz или 32768 Hz в зависимости от целевого железа.
- Диапазон громкости: 0.0–1.0 линейно или эквивалент в dB (например, -60 dB … 0 dB); весь микс в одной шкале.

Платформенные переопределения описываются в модуле аудио. Приоритет всегда у звуков, влияющих на геймплей.

## Динамика

Аудиосистема должна поддерживать изменение звука в зависимости от ситуации в игре.

Изменения включают:

- ускорение музыки при увеличении сложности
- изменение громкости при опасности
- временные звуки при бонусах
- специальные звуки при комбо
- отдельные сигналы для кооперативной игры

Музыка и эффекты усиливают обратную связь игроку:

- успех сопровождается чётким звуком
- ошибка сопровождается резким сигналом
- опасность слышна заранее

Звуки не мешают управлению и не снижают читаемость игрового процесса. При большом количестве одновременных событий система воспроизводит звуки в порядке приоритета; звуки, важные для геймплея, имеют наивысший приоритет.
